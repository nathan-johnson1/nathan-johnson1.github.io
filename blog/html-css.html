Best Practices Associated With Using Classes vs Ids

What Are IDs and Classes?

Think of them as name tags:

  • ID: A unique name for one specific element.
  • Class: A shared label you can use on multiple elements.

A Simple Example

Here’s what it looks like in code:

<div id="header">Welcome to My Website</div>
<div class="feature">Cool Feature 1</div>
<div class="feature">Cool Feature 2</div>

And the CSS that styles it:

#header {
  background-color: navy;
  color: white;
  padding: 20px;
}

.feature {
  background-color: lightgray;
  padding: 10px;
  margin-top: 10px;
}

What you'd see: A navy bar that says "Welcome to My Website", and two light gray boxes stacked underneath, both styled the same.

Best Practices

Use IDs when:

  • You’re labeling just one specific element
  • You need to anchor or link to it (e.g. #about-us)
  • It’s unique — like a header or footer

Example:

<div id="footer">Copyright 2025</div>

Use classes when:

  • You’re styling multiple elements the same way
  • You want consistency across repeated elements

Example:

<div class="card">Product 1</div>
<div class="card">Product 2</div>

Quick Comparison Table

Feature ID (#) Class (.)

Final Thoughts

Using IDs and classes correctly keeps your site organized and makes styling simpler. If you remember just one thing: use IDs for one-of-a-kind items, and classes for repeatable ones.
